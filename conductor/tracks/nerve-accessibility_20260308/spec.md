# Specification: Week 1 — The Nerve: AccessibilityService Core

**Track ID:** nerve-accessibility_20260308
**Type:** Feature
**Created:** 2026-03-08
**Status:** Draft

## Summary

Implement the foundational AccessibilityService layer (L2 PERCEPTION + L5 NERVE) that enables Neuron to read any Android app's UI tree and execute actions (tap, type, swipe, scroll) autonomously. Includes screenshot capture, floating overlay UI, and debug tooling.

## Context

Neuron is an AI-powered Android agent that controls phones via natural language. The Nerve layer is the first and most critical component — without reliable UI reading and action execution, nothing else works. This track implements Days 1-5 of the 3-week MVP sprint.

## User Story

As a Neuron user, I want the app to read and control any Android screen so that I can automate tasks through natural language without touching the phone. As a developer, I want a reliable AccessibilityService foundation so that the Brain layer (Week 2) can plan and execute actions on any app.

## Acceptance Criteria

- [ ] UITreeReader captures any app's UI tree as JSON in < 50ms
- [ ] ActionExecutor can tap, type, swipe, scroll, and perform global actions on any app
- [ ] ScreenCapture returns compressed JPEG screenshots
- [ ] OverlayManager shows draggable floating bubble with status animations and text input
- [ ] Debug overlay shows colored bounding boxes on all UI elements
- [ ] End-to-end: open WhatsApp → tap contact → type message → tap send (zero human interaction)
- [ ] Works on both Pixel (stock Android) and Samsung (OneUI)

## Dependencies

None — this is the foundation layer, first component to be built.

## Out of Scope

## Technical Notes

- Hilt for all DI — never manual construction
- Kotlin Coroutines for all async (never block main thread)
- Always recycle AccessibilityNodeInfo after use
- Compose-only UI (no XML layouts, no Fragments)
- API 26+ minimum, optimize for API 30+
- TYPE_ACCESSIBILITY_OVERLAY for overlay windows
